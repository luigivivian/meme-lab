# Archived Requirements: v1.0 — Auth, Rate Limiting & Gemini Image Fix

**Defined:** 2026-03-23
**Completed:** 2026-03-24
**Core Value:** O pipeline nunca para de gerar conteúdo — degrada graciosamente quando limites são atingidos

## v1 Requirements (25/25 Complete)

### Pre-Conditions

- [x] **PRE-01**: CORS configurado com origins específicos (não wildcard) para suportar credentials
- [x] **PRE-02**: Gemini Image model names validados via list_models() e corrigidos (fix 400)
- [x] **PRE-03**: API keys mascaradas em todos os logs (fix exposição existente)

### Authentication Backend

- [x] **AUTH-01**: Usuário pode criar conta com email e senha (bcrypt hash)
- [x] **AUTH-02**: Usuário pode fazer login e receber JWT access token + refresh token
- [x] **AUTH-03**: Usuário pode renovar access token usando refresh token
- [x] **AUTH-04**: Usuário pode fazer logout (invalidar refresh token)
- [x] **AUTH-05**: Todas as rotas da API protegidas por JWT (exceto login/registro/health)
- [x] **AUTH-06**: Sistema de roles (admin/user) com seed admin na migration
- [x] **AUTH-07**: Tabela users no MySQL (email, hashed_password, role, api_keys criptografadas, created_at)

### Authentication Frontend

- [x] **FAUTH-01**: Página de login com formulário email/senha no memeLab
- [x] **FAUTH-02**: Página de registro para criar conta nova
- [x] **FAUTH-03**: Middleware Next.js redirecionando rotas protegidas para login
- [x] **FAUTH-04**: AuthContext gerenciando tokens e estado de autenticação
- [x] **FAUTH-05**: api.ts injetando Authorization header em todas as requisições

### Quota Control

- [x] **QUOT-01**: Tabela api_usage no MySQL (user_id, service, tier, date, count, status)
- [x] **QUOT-02**: Tracking atômico de uso por usuário por dia (SELECT FOR UPDATE)
- [x] **QUOT-03**: Limites diários configuráveis via env vars (GEMINI_IMAGE_DAILY_LIMIT_FREE)
- [x] **QUOT-04**: Dual key management: key free como padrão, key paga como fallback
- [x] **QUOT-05**: UsageAwareKeySelector que resolve qual key usar baseado no consumo
- [x] **QUOT-06**: Fallback automático para backgrounds estáticos quando limite free atingido
- [x] **QUOT-07**: Reset diário do contador (timezone-aware)

### Usage Dashboard

- [x] **DASH-01**: Widget no dashboard mostrando consumo diário vs limite
- [x] **DASH-02**: Indicador visual de source usado (gemini/comfyui/static) por imagem
- [x] **DASH-03**: Endpoint API retornando estatísticas de uso do usuário
