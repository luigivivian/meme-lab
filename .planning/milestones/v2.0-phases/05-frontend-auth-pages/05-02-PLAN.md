**Component: `export default function LoginPage()`**

**State variables:**
- `email: string` (initially `""`)
- `password: string` (initially `""`)
- `showPassword: boolean` (initially `false`)
- `error: string` (initially `""`)
- `fieldErrors: Record<string, string>` (initially `{}`)
- `isSubmitting: boolean` (initially `false`)

**Get auth:** `const { login } = useAuth();`

**Client-side validation function `validate(): boolean`:**
- Reset `fieldErrors` to `{}`.
- If `email.trim() === ""`, set `fieldErrors.email = "Campo obrigatorio."`, return false.
- If email does not match `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`, set `fieldErrors.email = "Informe um email valido."`, return false.
- If `password === ""`, set `fieldErrors.password = "Campo obrigatorio."`, return false.
- Return true.

**Submit handler `handleSubmit(e: FormEvent)`:**
1. `e.preventDefault()`.
2. Clear `error` to `""`.
3. If `!validate()`, return.
4. Set `isSubmitting = true`.
5. Try: `await login(email, password)`.
   - Note: login() handles redirect to /dashboard internally via window.location.href.
6. Catch: set `error` from caught error. Map known messages:
   - If `err.message` includes "Invalid email or password" -> set `error = "Email ou senha incorretos."`.
   - If `err.message` includes "Failed to fetch" or is a TypeError -> set `error = "Sem conexao com o servidor. Verifique sua internet."`.
   - Otherwise -> set `error = "Erro no servidor. Tente novamente em alguns instantes."`.
7. Finally: set `isSubmitting = false`.
