## What Changed

### Google Trends Traffic Parsing (google_trends.py)
- **Before:** Char-by-char digit extraction, only checked for "K"/"M" presence loosely, gave 0.7 or 0.9
- **After:** `_parse_traffic()` function using `re.search(r'(\d+\.?\d*)\s*([KMBkmb]?)')` with multiplier map
- Score range: 0.4 (<1K) to 0.95 (10M+), with granular thresholds at 1K/10K/100K/1M/10M
- Handles edge cases: None, empty string, no suffix, decimal numbers like "1.5M+"

### Reddit Engagement Scoring (reddit_memes.py)
- **Before:** Every post scored with static `base_score` (from tuple config, was 0.4 before Plan 01)
- **After:** Two-pass approach:
  1. Extract upvotes from RSS content HTML via `re.search(r'(\d+)\s*points?')`
  2. Engagement formula: `base_score * (0.5 + 0.5 * (upvotes / max_upvotes))`
  3. Position-based fallback: `base_score - (position * 0.02)`, min `base_score * 0.5`

### TrendAggregator Scoring (aggregator.py)
- **Multi-source boost:** Tracks `source_counts` per normalized title. Boost = `min(1.0 + 0.2 * (sources-1), 2.0)`. Keeps highest base score during dedup (not additive).
- **Temporal decay:** `_apply_temporal_decay()` applies `e^(-age_hours/24)`:
  - Fresh (<2h): ~92% retained
  - 24h old: ~37% retained
  - 48h old: ~14% retained
- **Ordering:** boost applied first, then decay, then sort by final score
