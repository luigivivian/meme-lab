## Implementation Details

### LLM Theme Mapping (D-12)

- New `_llm_map_theme()` function in `curator.py` calls `generate()` with `tier="lite"` (Gemini flash-lite)
- Maps topic+humor_angle to best situacao_key from the full pool (built-in SITUACOES + themes.yaml)
- In-memory `_theme_cache` dict prevents duplicate LLM calls for same topic
- `_THEME_DESCRIPTIONS` provides rich context to LLM for each of 17 theme keys
- `_THEME_MAPPING_PROMPT` includes visual scene descriptions and Brazilian cultural rules

### Fallback Chain in _match_situacao_diversa

1. **LLM mapping** (primary) -- Gemini flash-lite cached call
2. **KEYWORD_MAP** (fallback) -- preserved for offline/error resilience
3. **Random from pool** (last resort) -- shuffled pool without repetition

### Meme Potential Filter (D-14)

- ANALYZER_PROMPT updated to include `"meme_potential": 1-5` in JSON schema
- OLLAMA_SINGLE_PROMPT updated similarly
- `_selections_to_topics()` filters out topics with `meme_potential < 3`
- Safety fallback: if ALL topics filtered, keeps original selections

### Throughput Increase (D-13)

- Curator `curate()` default count: 5 -> 15
- Orchestrator drains 30 trends (up from 20)
- `curator_count = max(self.images_per_run * 3, 15)` replaces `min(self.images_per_run, len(events))`
