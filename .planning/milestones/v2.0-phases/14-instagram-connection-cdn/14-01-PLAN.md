Create `src/services/instagram_oauth.py` — InstagramOAuthService class:

```python
class InstagramOAuthService:
    """Handles Facebook OAuth flow for Instagram Business Account connection.

    Flow per D-04:
    1. generate_auth_url() -> Facebook OAuth URL with state CSRF token
    2. exchange_code(code, state) -> short-lived token -> long-lived token -> encrypted in DB
    3. refresh_token(connection_id) -> refresh before 60-day expiry
    4. disconnect(user_id) -> revoke token, mark disconnected
    5. get_status(user_id) -> connection status
    """
```

Key methods:

1. `__init__(self, session: AsyncSession)` — stores session reference

2. `generate_auth_url(self) -> dict` — per D-07:
   - Generate random `state` string (32 chars, secrets.token_urlsafe)
   - Build Facebook OAuth URL: `https://www.facebook.com/{FACEBOOK_GRAPH_API_VERSION}/dialog/oauth?client_id={FACEBOOK_APP_ID}&redirect_uri={FACEBOOK_OAUTH_REDIRECT_URI}&scope=instagram_basic,instagram_content_publish,pages_show_list,pages_read_engagement&state={state}&response_type=code`
   - Return `{"auth_url": url, "state": state}` — caller (frontend or API) must store state in session/cookie for CSRF validation

3. `exchange_code(self, code: str, user_id: int) -> InstagramConnection` — per D-08:
   - Step 1: Exchange code for short-lived token via `POST {FACEBOOK_GRAPH_API_BASE}/oauth/access_token` with params: client_id, redirect_uri, client_secret, code
   - Step 2: Exchange short-lived for long-lived token via `GET {FACEBOOK_GRAPH_API_BASE}/oauth/access_token?grant_type=fb_exchange_token&client_id={...}&client_secret={...}&fb_exchange_token={short_token}`
   - Step 3: Get user's Facebook Pages via `GET {FACEBOOK_GRAPH_API_BASE}/me/accounts?access_token={long_token}` — find page with `instagram_business_account`
   - Step 4: Get IG user info via `GET {FACEBOOK_GRAPH_API_BASE}/{ig_user_id}?fields=id,username&access_token={long_token}`
   - Step 5: Encrypt long-lived token with `_encrypt_token()`
   - Step 6: Upsert InstagramConnection (delete existing for user_id if any, then create new)
   - Step 7: Commit and return the connection object
   - Use httpx.AsyncClient for all HTTP calls with timeout=30s

4. `refresh_expiring_tokens(self, days_before_expiry: int = 7) -> int` — per D-05:
   - Query all connections where status='active' AND token_expires_at < now + days_before_expiry
   - For each: decrypt token, call `GET {FACEBOOK_GRAPH_API_BASE}/oauth/access_token?grant_type=fb_exchange_token&...`
   - Update encrypted token and new expiry (60 days from now)
   - Return count of refreshed tokens
   - On error: set connection status='error'

5. `get_status(self, user_id: int) -> dict | None` — per D-09:
   - Query InstagramConnection for user_id
   - Return `{connected: bool, ig_username, status, token_expires_at, connected_at}` or None

6. `disconnect(self, user_id: int) -> bool` — per D-10:
   - Find connection for user_id
   - Set status='disconnected', clear access_token_encrypted
   - Return True if found and disconnected

7. `_encrypt_token(self, plaintext: str) -> str` — Fernet encryption:
   - Use `cryptography.fernet.Fernet` with key from INSTAGRAM_TOKEN_ENCRYPTION_KEY
   - If INSTAGRAM_TOKEN_ENCRYPTION_KEY is empty, derive from env SECRET_KEY (or JWT secret) using HKDF
   - Actually, keep it simple: if INSTAGRAM_TOKEN_ENCRYPTION_KEY is empty, use `Fernet.generate_key()` logged once as warning — but for production, user must set it
   - Encrypt and return base64 string

8. `_decrypt_token(self, ciphertext: str) -> str` — Fernet decryption

Add `cryptography` to requirements if not already present (it likely is since the project uses JWT).

Error handling: Wrap Facebook API calls in try/except, raise `InstagramOAuthError(message, fb_error_code)` custom exception class defined at top of file.

Logging: Use `logger = logging.getLogger("clip-flow.instagram-oauth")`.
