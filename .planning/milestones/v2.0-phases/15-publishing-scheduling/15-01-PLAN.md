Replace the placeholder `_publish_instagram()` method in PublishingService with real Instagram Graph API integration per D-03 and D-17.

The publishing flow must be:
1. Look up the user_id for this post. The ScheduledPost has character_id. Load the Character to get user_id. If no character_id, check content_package -> pipeline_run -> character -> user_id chain.
2. Load the active InstagramConnection for that user_id from DB (query InstagramConnection where user_id=X and status='active'). If none found, raise a descriptive error "No active Instagram connection for user {user_id}".
3. Decrypt the access_token using InstagramOAuthService._decrypt_token (instantiate the service with self.session).
4. Get the ig_user_id from the InstagramConnection record (this is the Instagram business_id).
5. Load the ContentPackage for this post to get image_path and caption.
6. Upload the image to GCS using GCSUploader(bucket_name="meme-lab-bucket") with remote_name=f"instagram-media/{package.id}/{filename}". This returns a signed public URL.
7. Build caption from package.caption if non-empty, else package.phrase. Append package hashtags as space-separated string.
8. Instantiate InstagramClient(access_token=decrypted_token, business_id=ig_user_id).
9. If package has carousel_slides with 2+ entries, use publish_carousel(). Otherwise use publish_image(image_url=signed_url, caption=caption_text).
10. Close the InstagramClient after publishing.
11. Return the result dict from InstagramClient (contains id, permalink, timestamp).

For error handling:
- Catch InstagramAPIError and re-raise as a regular Exception with the error message (so _mark_failed captures it).
- Catch InstagramRateLimitError specifically and set a longer retry window in the error message.
- Use try/finally to ensure InstagramClient.close() is called.

Also update `_mark_failed` to use exponential backoff: when a post fails and still has retries, set its `scheduled_at` to now + (retry_count * 120) seconds so it doesn't retry immediately. Import `timedelta` from datetime for this.
