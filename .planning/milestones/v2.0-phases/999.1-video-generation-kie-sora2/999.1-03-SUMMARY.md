## Decisions Made
- Background task uses get_session_factory() to create independent DB sessions since the request session is not available in background context
- Budget enforcement is a two-step process: pre-check estimated cost before starting, track actual cost via UsageRepository.increment() after completion
- GCS blob cleanup after video generation is best-effort and non-blocking to avoid failing the overall task on cleanup errors
- Theme key for motion prompt is resolved from image_metadata (theme_key or situacao_key fields) with fallback to "generico"
