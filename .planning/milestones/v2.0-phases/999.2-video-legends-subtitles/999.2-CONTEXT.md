# Phase 999.2: Video Legends & Subtitles - Context

**Gathered:** 2026-03-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Add text overlays (meme phrase + watermark) to generated videos using FFmpeg, matching the existing Pillow text style from image_maker.py. Produces ready-to-publish Instagram Reels with burned-in captions. Three animation modes: static, fade, typewriter. Auto-triggers after video generation completes.

</domain>

<decisions>
## Implementation Decisions

### Text Rendering Fidelity
- **D-01:** Match the exact Pillow style from image_maker.py — white text (255,255,255), black stroke (2px), shadow. FFmpeg `drawtext` with `borderw=2`, `bordercolor=black`, `shadowcolor`.
- **D-02:** No overlay/vignette/glow effects on video — Sora 2 video has its own visual style. Only text + watermark overlay. Per 999.1 D-01, the raw video is the canvas.
- **D-03:** Use the same font file from `fonts/` directory (`.ttf`). FFmpeg `fontfile` param takes the same path. Consistent typography across images and videos.
- **D-04:** Text position at bottom area (vertical 0.80) matching image_maker.py. Recognizable Mago Mestre brand format.

### Animation Modes & Defaults
- **D-05:** Default mode is `static` — text visible for full video duration. Always readable, simplest. `VIDEO_LEGEND_MODE=static`.
- **D-06:** Fade mode uses fade-in only (~0.5s). Text appears and stays visible. Always readable after reveal.
- **D-07:** Typewriter mode at ~30 chars/sec — approximately 2 seconds for a typical meme phrase. Natural reading pace.
- **D-08:** Global default mode via `VIDEO_LEGEND_MODE` env var + per-video override via API parameter. Flexibility without complexity.

### Pipeline & File Management
- **D-09:** Auto-trigger after video generation completes — when `video_status='completed'`, legend worker processes automatically. Seamless pipeline.
- **D-10:** New output file with `_legend` suffix — keep original video for re-processing with different modes.
- **D-11:** Graceful fallback on FFmpeg failure — video without text is still usable. Log error, don't block pipeline.
- **D-12:** 10% vertical safe margin for Instagram Reels — avoids IG UI elements (username, likes, caption overlay area).

### Claude's Discretion
- FFmpeg filter chain ordering and exact drawtext parameters
- Word-wrap algorithm adaptation from Pillow to FFmpeg character widths
- Typewriter animation implementation (char-by-char vs word-by-word)
- Legend worker async integration pattern within PostProductionLayer
- Database column additions for legend status tracking (if needed)
- Exact fade timing curves

</decisions>

<specifics>
## Specific Ideas

- Port `_wrap_text()` from image_maker.py but adapt for FFmpeg's character width estimation (monospace approximation or pre-calculate with Pillow)
- Watermark at bottom-right matching image_maker.py style (22pt, WATERMARK_COLOR)
- Use `ffmpeg-python` library for clean filter chain construction (not raw subprocess)
- Output format: MP4 H.264 (Instagram Reels compatible), same resolution as input video

</specifics>

<deferred>
## Deferred Ideas

- **Animated watermark** — rotating or pulsing watermark effect. Adds complexity, defer to future enhancement.
- **Multi-language subtitles** — SRT-based subtitle tracks for different languages. Requires translation pipeline.
- **Dynamic font sizing** — auto-adjust font size based on phrase length. Current fixed size is sufficient for typical meme phrases.

</deferred>
