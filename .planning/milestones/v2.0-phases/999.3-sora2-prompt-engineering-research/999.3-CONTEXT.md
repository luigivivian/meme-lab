<decisions>
## Implementation Decisions

### Research & Template Strategy
- **D-01:** Research scope focuses on **image-to-video** techniques only — filter awesome-sora2 for i2v patterns (camera control, temporal consistency, subject animation). Skip text-to-video-only techniques.
- **D-02:** Update **all 17 MOTION_TEMPLATES** — consistent quality across all themes. v1 templates preserved via config fallback.
- **D-03:** Prompt improvements focus on **camera movement + temporal flow + physics** — the three areas most impactful for i2v quality per community reports.
- **D-04:** Expand templates from 3 sentences to **4-5 sentences** — add camera direction and physics/cloth detail. Research shows more specific prompts yield better i2v results.

### Versioning & System Prompt
- **D-05:** Version switching via **`VIDEO_PROMPT_STYLE` env var** — `v1` returns current templates unchanged, `v2` returns researched templates. Default to `v2`.
- **D-06:** **Restructure `_SYSTEM_PROMPT` and `_ENHANCE_PROMPT`** with Sora 2-specific keywords — temporal sequencing vocabulary, camera movement taxonomy, physics/cloth terms from research findings.
- **D-07:** Increase prompt length cap from 300 to **500 chars** — research suggests medium-length prompts (300-500) work best for i2v.
- **D-08:** **Add technique reference comments** in code — future maintainers understand why each pattern was chosen.

### Claude's Discretion

- Specific camera movement vocabulary to use
- Temporal flow patterns (e.g., "slow motion entrance → steady state → subtle exit")
- System prompt structure and keyword ordering
- Test approach for comparing v1 vs v2 prompt quality
