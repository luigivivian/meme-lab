## Interaction States

### Input Fields

| State | Visual |
|-------|--------|
| Default | `bg-background border-input` (#06060a bg, #1a1a24 border) |
| Focus | `ring-2 ring-ring` (2px #8B5CF6 ring, offset 2px) |
| Error | `border-destructive` (#f43f5e border), red error text below |
| Disabled | `opacity-50 cursor-not-allowed` |

### Submit Button

| State | Visual |
|-------|--------|
| Default | `bg-primary text-white` (#8B5CF6 background), lg size (h-12 px-8), full width (`w-full`) |
| Hover | `bg-primary/90` + purple glow shadow |
| Active | `scale-[0.97]` (existing button behavior) |
| Loading | Button disabled, text replaced with spinner (16px Loader2 icon from lucide-react, `animate-spin`) + "Entrando..." / "Criando conta..." |
| Disabled | `opacity-50 pointer-events-none` |

### Form Validation

- **Approach:** Inline errors below each field (not toast)
- **Timing:** On submit only (not on blur for v1 simplicity)
- **Error display:** `<p className="text-sm text-destructive mt-1">` below the invalid field
- **Client-side checks:** Email format (basic regex), password min length (1 char -- backend enforces real rules), confirm password match
- **Server-side errors:** Displayed in a single error area above the submit button

### Password Field

- **Show/hide toggle:** Yes. Button with `Eye` / `EyeOff` lucide icon at `absolute right-3 top-1/2 -translate-y-1/2`, ghost variant, icon size 16px
- **Input type:** Toggles between `type="password"` and `type="text"`

## AuthContext Contract

```typescript
interface AuthContextType {
  user: { email: string; role: string } | null;
  isAuthenticated: boolean;
  isLoading: boolean;  // true during initial hydration from localStorage
  login: (email: string, password: string) => Promise<void>;
  register: (email: string, password: string) => Promise<void>;
  logout: () => Promise<void>;
}
```

- **Provider location:** `app/layout.tsx` wrapping `{children}`
- **Token storage:** localStorage keys `access_token`, `refresh_token` (D-01)
- **On mount:** Read `access_token` from localStorage. If present, call `/auth/me` to validate and populate `user`. If 401, clear tokens and set `user: null`.
- **login():** POST `/auth/login`, store both tokens, call `/auth/me`, redirect to `/dashboard`
- **register():** POST `/auth/register`, then call `login()` automatically (D-10)
- **logout():** POST `/auth/logout` (with current token), clear localStorage, redirect to `/login` (D-11)
- **On 401 anywhere:** Redirect to `/login` (D-02)

## API Header Injection Contract

Modify `api.ts` `request()` function:

```typescript
// Before each request, read token from localStorage
const token = typeof window !== "undefined"
  ? localStorage.getItem("access_token")
  : null;

const headers: Record<string, string> = {
  "Content-Type": "application/json",
  ...options?.headers,
};

if (token) {
  headers["Authorization"] = `Bearer ${token}`;
}
```

- **Source:** localStorage directly (D-08), not React context
- **SSR safety:** Guard with `typeof window !== "undefined"`

## Accessibility

| Concern | Implementation |
|---------|---------------|
| Focus management | First input (`email`) receives `autoFocus` on page load |
| Focus visible | Existing global style: 2px solid `--color-primary`, offset 2px |
| Form labels | Each input wrapped in `<label>` with visible text label above input |
| Error association | Error `<p>` elements use `id` matching input `aria-describedby` |
| Submit on Enter | Native form `<form onSubmit>` behavior |
| Screen reader | Error messages use `role="alert"` |
| Color contrast | #f0f0f5 on #06060a = 18.1:1 (AAA). #8888a0 on #06060a = 5.7:1 (AA). #f43f5e on #0f0f14 = 5.4:1 (AA). |
| Reduced motion | Existing `@media (prefers-reduced-motion)` disables all animations |
