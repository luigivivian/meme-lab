    ```python
    """Product Ad Pipeline — 8-step orchestrator for AI video ad generation.

    Steps (per D-21):
    1. analysis  — Gemini Vision analyzes product, suggests defaults
    2. scene     — rembg bg removal + Gemini scene composition
    3. prompt    — LLM generates cinematic video prompt
    4. video     — Kie.ai video generation (Wan 2.6 / Kling / Hailuo)
    5. copy      — LLM generates headline + CTA + hashtags
    6. audio     — Suno music + optional TTS narration
    7. assembly  — FFmpeg text overlay + audio mix + subtitles
    8. export    — Multi-format crop/pad with blur background

    Per D-20: Same approve/regenerate pattern as ReelsPipeline.
    Per D-22: Export is auto-complete (no approval needed).
    """
    ```

    Methods (all async except bg removal which uses asyncio.to_thread):

    1. `async def run_step_analysis(self, product_image_path: str) -> dict`:
       - Calls `analyze_product(product_image_path)`
       - Returns analysis dict (niche, tone, audience, scene_suggestions, product_description)

    2. `async def run_step_scene(self, product_image_path: str, scene_prompt: str, job_dir: str) -> str`:
       - Calls `remove_background(product_image_path, f"{job_dir}/cutout.png")` via asyncio.to_thread (CPU-bound)
       - Calls `compose_scene(cutout_path, scene_prompt, f"{job_dir}/scene.jpg")`
       - Returns scene image path

    3. `async def run_step_prompt(self, product_description: str, scene_description: str, style: str, video_model: str, tone: str) -> str`:
       - Calls `build_video_prompt(...)`
       - Returns generated prompt string

    4. `async def run_step_video(self, scene_image_path: str, prompt: str, video_model: str, job_dir: str, style: str) -> str | list[str]`:
       - Per D-09: single-shot for cinematic (1 clip), multi-scene for narrated/lifestyle
       - For single-shot: upload scene image via GCSUploader, call KieSora2Client.create_task, poll, download
       - For multi-scene: generate N clips (STYLE_SCENE_COUNT[style]) with varying prompts
       - Returns video path(s)

    5. `async def run_step_copy(self, product_name: str, product_description: str, niche: str, tone: str, audience: str, style: str) -> dict`:
       - Calls `generate_copy(...)`
       - Returns {"headline": str, "cta": str, "hashtags": list[str]}

    6. `async def run_step_audio(self, tone: str, audio_mode: str, job_dir: str, narration_text: str | None = None, video_duration: float = 15.0) -> dict`:
       - Per D-14: Generate music via KieMusicClient using MUSIC_MAP[tone]
       - If audio_mode == "narrated": also generate TTS via existing tts.py
       - Mix via format_exporter.mix_audio
       - Returns {"music_path": str, "tts_path": str | None, "mixed_path": str | None}

    7. `async def run_step_assembly(self, video_path: str, headline: str | None, cta: str | None, audio_path: str | None, style: str, job_dir: str, srt_path: str | None = None) -> str`:
       - Per D-16: overlay_text (if headline/cta)
       - Attach audio to video via FFmpeg if audio_path exists
       - If narrated + srt_path: burn subtitles (reuse video_builder pattern)
       - Returns assembled video path

    8. `async def run_step_export(self, assembled_path: str, job_dir: str, formats: list[str] | None = None) -> dict`:
       - Per D-18, D-22: auto-complete, no approval
       - Calls export_all_formats(assembled_path, job_dir, formats)
       - Returns dict mapping format to output path

    Also add:
    - `def estimate_cost(self, style: str, audio_mode: str, formats: list[str]) -> dict`: Per D-19, calculate estimated cost in BRL before generation. Returns {"video_brl": float, "audio_brl": float, "image_brl": float, "total_brl": float} using design doc estimates.

    Constructor: `def __init__(self, config: dict | None = None)` — stores config dict, lazy-inits clients.

    Use lazy imports inside step methods to avoid importing heavy deps at module level (same pattern as ReelsPipeline).
