    Create `src/api/routes/ads.py` following the reels.py pattern line-by-line:

    ```python
    """Product Ad pipeline API routes — wizard create, step execution, approve, regenerate.

    Per D-20: Same approve/regenerate pattern as reels.
    Per D-21: 8 steps: analysis, scene, prompt, video, copy, audio, assembly, export.
    Per D-22: Export is auto-complete (no approval needed).
    Per D-23: Separate /ads section.
    """
    router = APIRouter(prefix="/ads", tags=["Product Ads"])
    ```

    Endpoints (mirror reels.py exactly, adjusted for ads):

    1. `POST /ads/create` — Create new ad job
       - Accepts AdCreateRequest body
       - Creates ProductAdJob row with status="draft", generates UUID job_id
       - Initializes step_state with all 8 steps as {"status": "pending"}
       - Per D-19: Returns job with cost_estimate included
       - Returns AdJobResponse

    2. `GET /ads/jobs` — List user's ad jobs
       - Filter by user_id (tenant isolation)
       - Optional status filter query param
       - Order by created_at desc
       - Returns list[AdJobResponse]

    3. `GET /ads/{job_id}` — Get job detail
       - Returns AdJobResponse with full step_state
       - 404 if not found, 403 if wrong user

    4. `GET /ads/{job_id}/steps` — Get step state
       - Returns step_state dict with current_step and progress_pct

    5. `POST /ads/{job_id}/step/{step_name}/execute` — Execute a step
       - Validates step_name is in ADS_STEP_ORDER
       - Validates previous step is approved (or step is first)
       - Runs step via BackgroundTasks with get_session_factory()
       - Background task: update step_state[step_name].status to "generating", run pipeline step, update to "completed" with artifacts, or "failed" with error
       - Per D-22: If step is "export", auto-approve after completion
       - Uses flag_modified(job, "step_state") before every commit

    6. `POST /ads/{job_id}/step/{step_name}/approve` — Approve a step
       - Sets step_state[step_name].status = "approved"
       - Uses flag_modified
       - Returns updated step_state

    7. `POST /ads/{job_id}/step/{step_name}/regenerate` — Regenerate a step
       - Clears this step and all downstream steps (set to "pending")
       - Triggers execute for this step
       - Uses flag_modified

    8. `GET /ads/{job_id}/cost-estimate` — Cost estimate
       - Reads job config (style, audio_mode, output_formats)
       - Calls pipeline.estimate_cost()
       - Returns AdCostEstimate

    9. `GET /ads/{job_id}/file/{filename}` — Serve artifact file
       - Serves files from job output directory
       - FileResponse with proper content-type

    10. `DELETE /ads/{job_id}` — Delete job
        - Removes job row and output directory
        - 403 if wrong user

    Background task helper `_execute_ad_step_task`:
    - Same pattern as reels `_execute_step_task`
    - Uses get_session_factory() for independent DB sessions
    - Instantiates ProductAdPipeline with job config
    - Maps step_name to the corresponding run_step_* method
    - Stores step result in step_state[step_name]["result"]
    - Updates progress_pct based on step position in ADS_STEP_ORDER
    - Catches exceptions and sets step status to "failed" with error_message

    All endpoints must use `get_current_user` for tenant isolation and filter by user_id.
