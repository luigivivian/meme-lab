**Class KieSora2Client:**
- Logger: `logging.getLogger("clip-flow.kie_client")`
- Constructor: `__init__(self, api_key: str | None = None)` — reads from param or `config.KIE_API_KEY`. Raises `ValueError` if empty.
- Uses `httpx.AsyncClient` (NOT requests — project uses httpx for async HTTP per CONVENTIONS).
- Base URL: `https://api.kie.ai/api/v1`
- Headers: `{"Authorization": f"Bearer {api_key}", "Content-Type": "application/json"}`

**Methods:**

`async def create_task(self, image_url, prompt, duration=10, character_ids=None, aspect_ratio="portrait") -> str`:
- Build payload per research section 2:
  ```python
  payload = {
      "model": config.VIDEO_MODEL,  # default "sora-2-image-to-video"
      "input": {
          "prompt": prompt,
          "image_urls": [image_url],
          "aspect_ratio": aspect_ratio,
          "n_frames": str(duration),
          "remove_watermark": True,
          "upload_method": "s3",
          "character_id_list": character_ids or [],
      },
  }
  ```
- POST to `{BASE_URL}/jobs/createTask`
- Check `response.status_code` and `result["code"] == 200`
- Handle errors: 401 (auth), 402 (credits), 422 (validation), 429 (rate limit), 500+ (server)
- Return `result["data"]["taskId"]`

`async def get_task_status(self, task_id: str) -> dict`:
- GET `{BASE_URL}/jobs/recordInfo` with params `{"taskId": task_id}`
- Return `result["data"]` dict with state, progress, resultJson, failCode, failMsg, costTime

`async def poll_until_complete(self, task_id, initial_interval=None, max_interval=None, timeout=None) -> VideoGenerationResult | None`:
- Use config values as defaults: `KIE_POLL_INITIAL_INTERVAL`, `KIE_POLL_MAX_INTERVAL`, `KIE_POLL_TIMEOUT`
- Exponential backoff: `interval = min(interval * 1.5, max_interval)`
- Loop: `await asyncio.sleep(interval)`, call `get_task_status`, check state
- On `state == "success"`: `json.loads(data["resultJson"])`, extract `resultUrls[0]`
- Build `VideoGenerationResult` with video_url, task_id, generation_time_ms=data["costTime"]
- On `state == "fail"`: log `failMsg`, return None
- On timeout: log error, return None
- States to keep polling: `waiting`, `queuing`, `generating`

`async def download_video(self, video_url: str, output_path: str) -> str`:
- Use httpx to stream-download the MP4 file
- Create parent directory if not exists
- Write bytes to output_path
- Return output_path
- Log file size after download

`async def generate_video(self, image_url, prompt, duration=10, character_ids=None, output_dir=None) -> VideoGenerationResult | None`:
- High-level convenience method combining create_task + poll + download
- Calculate cost: `duration * config.VIDEO_COST_PER_SECOND`
- Generate output filename: `f"{task_id}.mp4"` in `output_dir or config.GENERATED_VIDEOS_DIR`
- Call `create_task` -> `poll_until_complete` -> `download_video`
- Set `result.local_path` and `result.cost_usd`
- Return result or None on failure

Use `async with httpx.AsyncClient(timeout=30) as client:` for each request, or store as instance variable with `__aenter__`/`__aexit__` context manager pattern for reuse.

`def build_motion_prompt(self, theme_key: str, phrase_context: str = "", pose: str = "", scene: str = "", video_prompt_notes: str = "") -> str`:

- Call `src.llm_client.generate()` with tier="lite" (per D-02: cost ~$0.001/prompt)
- System prompt:
  ```
  You generate SHORT motion/animation prompts for Sora 2 image-to-video AI.
  You receive context about a wizard character meme and must describe the MOTION and CAMERA movement.

  RULES:
  - Output ONLY the motion prompt (1-3 sentences, max 200 chars)
  - Describe character MOVEMENT (gestures, expressions, body language)
  - Describe CAMERA movement (slow zoom, pan, static, push in)
  - Describe AMBIENT motion (particles, wind, magical effects)
  - Keep the cartoon cel-shading art style
  - Portrait 4:5 aspect ratio composition
  - NEVER describe the character's appearance (that's in the image)
  - NEVER include text or dialogue
  ```
- User message includes:
  - `Base motion idea: {base_motion_template}`
  - `Theme: {theme_key}`
  - `Phrase context: {phrase_context}` (per D-02: motion relates to meme content)
  - `Character pose: {pose}` (if available)
  - `Scene description: {scene}` (if available)
  - `Improvement notes: {video_prompt_notes}` (per D-03: accumulated learnings)
- Parse LLM response: strip quotes, limit to 500 chars, fallback to base motion template if LLM fails
- Log the generated prompt
- Return the motion prompt string
