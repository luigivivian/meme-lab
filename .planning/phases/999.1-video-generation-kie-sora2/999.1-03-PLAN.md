**Helper functions:**

`async def _check_video_enabled()`:
- If not `VIDEO_ENABLED`: raise HTTPException(400, "Video generation is disabled. Set VIDEO_ENABLED=true in .env")
- If not `KIE_API_KEY`: raise HTTPException(400, "KIE_API_KEY not configured")

`async def _check_budget(session: AsyncSession, user_id: int, duration: int) -> float`:
- Calculate estimated cost: `duration * VIDEO_COST_PER_SECOND`
- Query `api_usage` for today's total cost where `service="kie_video"`
- Use `UsageRepository(session)` and `get_daily_usage(user_id, "kie_video", "standard")` (may need to add a method or use raw query)
- If `spent_today + estimated_cost > VIDEO_DAILY_BUDGET_USD`: raise HTTPException(429, f"Daily video budget exhausted. Spent ${spent_today:.2f} of ${VIDEO_DAILY_BUDGET_USD:.2f}")
- Return estimated_cost

`async def _generate_video_task(content_package_id: int, duration: int, character_ids: list[str], user_id: int, db_url: str)`:
- This is the **background task** function (runs after response returns, per D-06)
- Create a new async DB session (cannot reuse request session in background)
- Load ContentPackage by ID
- Load the Theme for this package's work_order.situacao_key (or from image_metadata)
- Get theme's `video_prompt_notes` (per D-03)
- Build motion prompt via `VideoPromptBuilder().build_motion_prompt(theme_key, phrase_context=pkg.phrase, video_prompt_notes=theme_notes)`
- Upload background image to GCS via `GCSUploader().upload_image(pkg.background_path)`
- Call `KieSora2Client().generate_video(image_url=signed_url, prompt=motion_prompt, duration=duration, character_ids=character_ids)`
- On success:
  - Update ContentPackage: `video_status="success"`, `video_path=result.local_path`, `video_source="kie_sora2"`, `video_prompt_used=result.prompt_used`, `video_task_id=result.task_id`, `video_metadata={"cost_usd": result.cost_usd, "duration": duration, "model": result.model, "generation_time_ms": result.generation_time_ms}`
  - Track cost: `UsageRepository(session).increment(user_id=user_id, service="kie_video", tier="standard", cost_usd=result.cost_usd)`
  - Cleanup: `GCSUploader().delete_blob(blob_name)` (optional, non-blocking)
- On failure:
  - Update ContentPackage: `video_status="failed"`, `video_metadata={"error": str(e)}`
- Commit session, close session

**Endpoints:**

`POST /generate/video` (single):
- Depends: `get_current_user`, `db_session`
- Body: `VideoGenerateRequest`
- Steps:
  1. `_check_video_enabled()`
  2. Load ContentPackage by ID, verify ownership (user_id match or admin), verify `approval_status == "approved"` (per D-05: only approved packages)
  3. If `video_status == "generating"`: raise HTTPException(409, "Video generation already in progress")
  4. Validate duration is 10 or 15 (per D-08)
  5. `_check_budget(session, user_id, duration)`
  6. Update ContentPackage: `video_status = "generating"`, commit
  7. Add `_generate_video_task` to `BackgroundTasks` (per D-06)
  8. Return `VideoStatusResponse` with `video_status="generating"`

`POST /generate/video/batch`:
- Depends: `get_current_user`, `db_session`, `BackgroundTasks`
- Body: `VideoBatchRequest`
- Steps:
  1. `_check_video_enabled()`
  2. Validate all content_package_ids exist and are owned by user and approved
  3. Check budget for total: `len(ids) * duration * VIDEO_COST_PER_SECOND`
  4. Mark all as `video_status="generating"`, commit
  5. Add one background task per package
  6. Return list of `VideoStatusResponse`

`GET /generate/video/status/{content_package_id}`:
- Depends: `get_current_user`, `db_session`
- Load ContentPackage, verify ownership
- Return `VideoStatusResponse` with current video_status, video_task_id, video_path, video_source, video_metadata

`GET /generate/video/budget`:
- Depends: `get_current_user`, `db_session`
- Query today's kie_video usage cost from `api_usage`
- Return `VideoBudgetResponse` with daily_budget, spent_today, remaining, videos_remaining_estimate (remaining / cost_per_video)
