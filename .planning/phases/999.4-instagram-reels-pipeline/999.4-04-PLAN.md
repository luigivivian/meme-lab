Create `src/api/routes/reels.py` following the video.py pattern (per D-01: expose via /reels/* routes).

**Router**: `APIRouter(prefix="/reels", tags=["Reels Pipeline"])`

**Endpoints:**

1. **POST /reels/generate** — Start pipeline as background task:
   - Accepts ReelGenerateRequest body
   - Creates ReelsJob row in DB (status="queued", generate job_id via uuid.uuid4().hex[:16])
   - If config_id provided, loads ReelsConfig from DB for that user
   - Builds config_override dict from DB config merged with request params
   - Calls `background_tasks.add_task(_generate_reel_task, job_id, req, config_override, session_factory)`
   - Returns `{"job_id": job_id, "status": "queued"}`
   - Uses `get_session_factory()` for background task DB access (per Phase 999.1 decision)

2. **GET /reels/status/{job_id}** — Poll status:
   - Query ReelsJob by job_id + user_id (tenant isolation)
   - Return ReelStatusResponse with status, current_step, progress_pct, video_url, error_message
   - 404 if not found

3. **GET /reels/jobs** — List user's reel jobs:
   - Query ReelsJob filtered by user_id, ordered by created_at DESC
   - Optional query params: status (str), limit (int, default=20), offset (int, default=0)
   - Return list of ReelJobResponse

4. **GET /reels/config** — Get user's config:
   - Query ReelsConfig by user_id (return all configs for user)
   - Return list of ReelsConfigResponse

5. **POST /reels/config** — Create/update config:
   - Accept ReelsConfigRequest body
   - Upsert by user_id + character_id + name (unique constraint)
   - Return ReelsConfigResponse

6. **GET /reels/config/presets** — List available presets:
   - Return static dict of 3 presets: "clean" (minimal, fast cuts), "bold" (large subtitles, slow transitions), "minimal" (defaults)
   - Per D-04: presets are pre-configured, editable

**Background task function `_generate_reel_task(job_id, req, config_override, session_factory)`:**
- Creates own DB session via session_factory (per Phase 999.1 pattern)
- Updates ReelsJob status to "generating"
- Creates ReelsPipeline(config_override=config_override)
- Defines on_progress callback that updates ReelsJob.current_step and progress_pct
- Calls `await pipeline.run(tema=req.tema, character_id=req.character_id, on_progress=on_progress)`
- On success: updates job with video_path, video_url (if GCS upload), script_json, caption, hashtags, cost_usd, cost_brl, status="complete"
- On exception: updates job status="failed", error_message=str(e), logs traceback

All endpoints use `Depends(get_current_user)` and `Depends(db_session)` for auth + tenant isolation.
