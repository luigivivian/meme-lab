# Phase 999.4: Instagram Reels Pipeline - Context

**Gathered:** 2026-03-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Pipeline **standalone** para geracao automatizada de Instagram Reels: imagens → roteiro multimodal → narracao TTS → legendas → montagem FFmpeg → MP4 pronto para publicar. Separado do pipeline multi-agente existente (src/pipeline/), mas reutilizando recursos ja integrados (Gemini API, FFmpeg legend_renderer, Instagram client).

</domain>

<decisions>
## Implementation Decisions

### D-01: Arquitetura — Pipeline Separada
- Pipeline vive em `src/reels_pipeline/` como modulo independente
- NAO integrar no pipeline multi-agente existente (src/pipeline/)
- Reutilizar modulos ja existentes: Gemini API client, GCS uploader, legend_renderer (FFmpeg), instagram_client
- Orquestrador proprio (`src/reels_pipeline/main.py`) com etapas sequenciais
- Expor via API FastAPI em rotas `/reels/*` separadas

### D-02: Providers — Reutilizar Stack Existente + Melhores Opcoes
- **Imagens:** Gemini Image API (ja integrado, `src/image_gen/gemini_client.py`) — reutilizar diretamente. Aspect ratio 9:16 vertical para Reels
- **Roteiro:** Gemini API (`google-genai`, ja integrado) — multimodal, analisa imagens geradas + gera roteiro JSON estruturado. Sem custo extra de API key
- **TTS:** OpenAI TTS-HD (`tts-1-hd`) como default — melhor qualidade PT-BR, custo baixo (~$0.004/30s). ElevenLabs como alternativa premium (voz clonada)
- **Transcricao:** OpenAI Whisper API como default (ja tem OPENAI_API_KEY no .env). AssemblyAI como alternativa. Whisper local (custo zero) como fallback
- **Montagem:** FFmpeg via subprocess — reutilizar patterns do `legend_renderer.py` existente
- **Upload/Publicacao:** Reutilizar `instagram_client.py` existente e GCS uploader

### D-03: Roteiro via Gemini API
- Gemini multimodal recebe as imagens geradas + prompt de roteiro
- Retorna JSON estruturado: titulo, gancho, cenas[], narracao_completa, CTA, hashtags, caption
- Template de system prompt configuravel por character/nicho
- Lingua: PT-BR default, configuravel

### D-04: Configuracao com Painel
- Modelo padrao funcional out-of-the-box (zero config para comecar)
- Painel de ajustes finos no dashboard (nova pagina `/reels` ou section em `/settings`)
- Configuracoes persistidas no banco (tabela `reels_config` por user/character)
- Parametros configuraveis:
  - **Imagens:** quantidade (3-10), estilo, aspect ratio
  - **Roteiro:** tom (inspiracional/humor/educativo), duracao (15/30/60s), nicho, CTA default, keywords
  - **TTS:** provider (openai/elevenlabs), voz, velocidade (0.8-1.5x)
  - **Transcricao:** provider (openai/assemblyai/whisper_local)
  - **Video:** duracao por imagem, tipo de transicao (crossfade/cut), musica de fundo (on/off + volume), estilo de legenda (posicao, fonte, cor), logo/watermark
- Presets: "Clean", "Bold", "Minimal" (pre-configurados, editaveis)

### Claude's Discretion
- Estrutura interna dos modulos (classes vs funcoes)
- Schema exato da tabela reels_config
- Ordem de fallback entre providers
- Formato do progresso/polling durante geracao
- Estrategia de cache de imagens entre runs

</decisions>
