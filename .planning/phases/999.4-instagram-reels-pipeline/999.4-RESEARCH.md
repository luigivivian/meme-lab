# Phase 999.4: Instagram Reels Pipeline - Research

**Researched:** 2026-03-27
**Domain:** Automated video pipeline (image slideshow + TTS + subtitles + FFmpeg assembly)
**Confidence:** MEDIUM-HIGH

## Summary

This phase builds a standalone pipeline that chains 5 modules: image generation (reusing existing GeminiImageClient), script generation (Gemini multimodal API with structured JSON output), TTS narration (OpenAI tts-1-hd), transcription to SRT (OpenAI Whisper API), and FFmpeg video assembly (slideshow with crossfade + audio + subtitles). The final output is an MP4 ready for Instagram Reels publishing via the existing InstagramClient.

The project already has mature infrastructure for most of this: GeminiImageClient for image gen, LegendRenderer for FFmpeg text overlays, GCSUploader for public URLs, InstagramClient with `publish_reel()` already implemented, FastAPI background task pattern from video.py, and the google-genai SDK (v1.68.0) already installed. The main new integrations are: OpenAI TTS API, OpenAI Whisper API for transcription, FFmpeg slideshow assembly (different from the existing legend overlay which works on pre-existing video), and Gemini structured JSON output for script generation.

**Primary recommendation:** Build as `src/reels_pipeline/` with 5 focused modules mirroring the reference doc structure, reuse existing clients directly (not wrapper-on-wrapper), and use OpenAI for both TTS and transcription to minimize API key sprawl.

### Pattern 1: Sequential Pipeline Orchestrator
**What:** Each step runs sequentially, passing artifacts to the next step. Each step saves intermediate files to a job directory.
**When to use:** Always for this pipeline (each step depends on previous output).
**Example:**
```python
# src/reels_pipeline/main.py
class ReelsPipeline:
    def __init__(self, config: ReelsConfig):
        self.config = config
        self.job_dir: Path = None

    async def run(self, tema: str, character_id: int, **kwargs) -> ReelsResult:
        self.job_dir = self._create_job_dir(tema)

        # Step 1: Generate images (9:16 vertical)
        image_paths = await self._generate_images(tema, character_id)

        # Step 2: Generate script via Gemini multimodal
        script = await self._generate_script(image_paths, tema)

        # Step 3: Generate narration via TTS
        audio_path = await self._generate_tts(script["narracao_completa"])

        # Step 4: Transcribe to SRT
        srt_path = await self._transcribe(audio_path)

        # Step 5: Assemble video via FFmpeg
        video_path = await self._build_video(image_paths, audio_path, srt_path)

        return ReelsResult(video_path=video_path, script=script)
```

### Pattern 3: FFmpeg Slideshow with Crossfade (xfade filter)
**What:** Create video from images with smooth crossfade transitions, then overlay audio and subtitles.
**When to use:** Module 5 video assembly.
**Example:**
```python
# Each image loaded separately with -loop 1 -t {duration}
# xfade filter chains between pairs
# Scale+pad to 1080x1920 for consistent 9:16
# Audio mixed with optional background music
# Subtitles via drawtext or ASS subtitles filter

def _build_xfade_command(image_paths, audio_path, srt_path, output_path, config):
    inputs = []
    for img in image_paths:
        inputs += ["-loop", "1", "-t", str(config.image_duration), "-framerate", "30", "-i", img]
    inputs += ["-i", audio_path]

    # Scale each image to 1080x1920
    scale_filters = []
    for i in range(len(image_paths)):
        scale_filters.append(
            f"[{i}]scale=1080:1920:force_original_aspect_ratio=decrease,"
            f"pad=1080:1920:-1:-1:color=black[s{i}]"
        )

    # Chain xfade transitions
    xfade_filters = []
    prev = "s0"
    for i in range(1, len(image_paths)):
        offset = i * config.image_duration - i * config.transition_duration
        out = f"f{i}" if i < len(image_paths) - 1 else "vout"
        xfade_filters.append(
            f"[{prev}][s{i}]xfade=transition=fade:duration={config.transition_duration}:offset={offset}[{out}]"
        )
        prev = f"f{i}" if i < len(image_paths) - 1 else "vout"

    # Add subtitle overlay
    sub_filter = f"[vout]subtitles={srt_path}:force_style='FontSize=52,PrimaryColour=&HFFFFFF,Outline=2,MarginV=200'[final]"

    filter_complex = ";".join(scale_filters + xfade_filters + [sub_filter])

    cmd = ["ffmpeg", "-y"] + inputs + [
        "-filter_complex", filter_complex,
        "-map", "[final]", "-map", f"{len(image_paths)}:a",
        "-c:v", "libx264", "-preset", "fast", "-crf", "18",
        "-c:a", "aac", "-b:a", "192k",
        "-r", "30", "-shortest", "-movflags", "+faststart",
        output_path,
    ]
    return cmd
```

### Anti-Patterns to Avoid
- **Wrapping existing clients in another wrapper:** Call GeminiImageClient directly, do not create ReelsImageClient that delegates to it.
- **Using concat demuxer for crossfade:** The concat demuxer does NOT support crossfade transitions. Must use xfade filter with separate inputs per image.
- **Parsing JSON from free-text LLM output:** Use Gemini's `response_schema` to guarantee valid JSON. The reference doc's approach of stripping markdown backticks is fragile.
- **Running FFmpeg synchronously in API thread:** Always use BackgroundTasks or run_in_executor.

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| TTS audio generation | Custom speech synthesis | OpenAI tts-1-hd API | PT-BR quality, $0.03/1K chars, simple API |

## Common Pitfalls

### Pitfall 1: FFmpeg xfade Requires Same Resolution
**What goes wrong:** xfade filter fails with "First input link parameters do not match" if images have different dimensions.
**Why it happens:** Gemini image generation may produce slightly different sizes, or aspect ratio enforcement may vary.
**How to avoid:** Always scale+pad each image to exact 1080x1920 before xfade: `scale=1080:1920:force_original_aspect_ratio=decrease,pad=1080:1920:-1:-1:color=black`
**Warning signs:** FFmpeg error mentioning "parameters do not match" or "xfade".

### Pitfall 2: OpenAI TTS 4096 Character Limit
**What goes wrong:** TTS API rejects text longer than 4096 characters.
**Why it happens:** A 60-second reel narration in PT-BR can approach this limit.
**How to avoid:** Validate narration length before TTS call. For 30s reels (target), ~150-200 words / ~1000 chars is typical, well within limit. For 60s, split into chunks if needed.
**Warning signs:** 400 error from OpenAI API.

### Pitfall 3: Whisper SRT vs verbose_json Tradeoff
**What goes wrong:** Using response_format="srt" gives segment-level timestamps (acceptable), but word-level alignment is not available with SRT format.
**Why it happens:** Whisper API only provides word-level timestamps with `verbose_json` + `timestamp_granularities=["word"]`.
**How to avoid:** For initial implementation, use response_format="srt" (simplest, per reference doc). If subtitle timing needs improvement later, switch to verbose_json and build SRT manually from word timestamps.
**Warning signs:** Subtitle text appearing slightly out of sync with narration.

### Pitfall 4: FFmpeg Not Installed on macOS
**What goes wrong:** subprocess.run("ffmpeg"...) raises FileNotFoundError.
**Why it happens:** FFmpeg is not currently installed on this machine (verified: `which ffmpeg` returns nothing).
**How to avoid:** Add FFmpeg availability check at pipeline startup. Document `brew install ffmpeg` as prerequisite. The existing LegendRenderer.is_available() pattern shows how to check.
**Warning signs:** FileNotFoundError or "command not found" in subprocess output.

### Pitfall 5: Gemini Image Gen Rate Limits (429)
**What goes wrong:** Generating 5-10 images for a single reel hits the 15 RPM / 500 RPD free tier limit.
**Why it happens:** Each image requires a separate API call to Gemini.
**How to avoid:** Existing GeminiImageClient already has retry logic for 429. Add configurable pause between image generations (existing PAUSA_SEGUNDOS pattern). For 5 images at 15 RPM, min 20s total (4s between calls).
**Warning signs:** RESOURCE_EXHAUSTED errors from Gemini API.

### Pitfall 6: OpenAI Python Package Not Installed
**What goes wrong:** `import openai` fails at runtime.
**Why it happens:** The openai Python package is not currently installed (only the npm package for frontend). Only google-genai is installed.
**How to avoid:** Add `openai` to requirements.txt and install before implementing TTS/Whisper modules.
**Warning signs:** ImportError on first run.

## Code Examples

### OpenAI TTS-HD for PT-BR Narration
```python
# Source: OpenAI TTS API docs
from openai import OpenAI

client = OpenAI()  # reads OPENAI_API_KEY from env

def generate_narration(text: str, output_path: str, voice: str = "nova", speed: float = 1.1) -> str:
    response = client.audio.speech.create(
        model="tts-1-hd",
        voice=voice,  # nova works well for PT-BR
        input=text,
        speed=speed,
    )
    response.stream_to_file(output_path)
    return output_path
```

### OpenAI Whisper API -> SRT
```python
# Source: OpenAI Speech-to-Text docs
from openai import OpenAI

client = OpenAI()

def transcribe_to_srt(audio_path: str, output_path: str, language: str = "pt") -> str:
    with open(audio_path, "rb") as f:
        result = client.audio.transcriptions.create(
            model="whisper-1",
            file=f,
            response_format="srt",
            language=language,
        )
    with open(output_path, "w", encoding="utf-8") as f:
        f.write(result)
    return output_path
```

### Gemini Multimodal Image Analysis + Structured JSON
```python
# Source: google-genai SDK docs + Gemini structured output docs
from google import genai
from google.genai import types
import json

def generate_script(client: genai.Client, image_paths: list[str], config: dict) -> dict:
    # Load images as Parts
    parts = []
    for img_path in image_paths:
        with open(img_path, "rb") as f:
            img_bytes = f.read()
        mime = "image/jpeg" if img_path.endswith(".jpg") else "image/png"
        parts.append(types.Part.from_bytes(data=img_bytes, mime_type=mime))

    # Add text prompt
    parts.append(ROTEIRO_PROMPT.format(**config))

    response = client.models.generate_content(
        model="gemini-2.5-flash",
        contents=parts,
        config={
            "response_mime_type": "application/json",
            "response_schema": ROTEIRO_SCHEMA,
            "system_instruction": SYSTEM_PROMPT_TEMPLATE.format(**config),
        },
    )
    return json.loads(response.text)
```
