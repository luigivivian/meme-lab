2. Add to `video_builder.py` three new functions per D-04:

`segment_roteiro(script: dict, max_segment_duration: float = REELS_SEGMENT_MAX_DURATION) -> list[dict]`:
- Split script["cenas"] into segments where each segment's total `duracao_segundos` <= max_segment_duration.
- Each segment dict: `{"cenas": [...], "duration": float, "index": int}`.
- If a single cena exceeds max_segment_duration, it goes alone in its own segment.

`build_segment_videos(segments: list[dict], image_paths_by_segment: list[list[str]], audio_path: str, srt_path: str, job_dir: str, config_override: dict | None = None) -> list[str]`:
- For each segment, slice audio using ffmpeg `-ss start -to end` based on cumulative cena durations.
- Slice SRT entries by timestamp range.
- Call `build_reel_video()` for each segment with its images + audio slice + SRT slice.
- Returns list of segment video paths.
- Audio slicing: `ffmpeg -y -i audio.wav -ss {start} -to {end} -c copy segment_audio_{i}.wav`
- SRT slicing: parse SRT file, filter entries within time range, reindex and adjust timestamps.

`concat_segments(segment_paths: list[str], output_path: str, transition_duration: float = 0.5) -> str`:
- Concatenate segment videos with xfade crossfade between them.
- Use the xfade chain pattern: `offset_i = sum(durations[0:i]) - i * transition_duration`.
- Get each segment duration via `get_video_duration()`.
- Returns final output_path.

Add helper `_parse_srt_time(time_str: str) -> float` and `_format_srt_time(seconds: float) -> str` for SRT timestamp conversion.
