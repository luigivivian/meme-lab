**2. Add step-based endpoints to `src/api/routes/reels.py`:**

Keep ALL existing endpoints. Add new ones:

`POST /reels/interactive` — Create interactive reel job. Same character resolution as existing `/generate` but sets `status="interactive"` and initializes `step_state` with `{"current_step": 0, "prompt": {"text": tema, "approved": false}, ...}`. Returns `{job_id, step_state}`.

`GET /reels/{job_id}/step-state` — Return full step_state JSON. Tenant-isolated.

`POST /reels/{job_id}/step/{step_name}` — Execute a step. Calls the corresponding `ReelsPipeline.run_step_*()` method in a background task. Updates step_state with artifacts on completion. Valid step_names: prompt, images, script, tts, srt, video.
- For "prompt": just saves text to step_state (no background task needed).
- For "images", "script", "tts", "srt", "video": runs in background task using `get_session_factory()` pattern. Updates job.step_state on completion. Sets step-level status to "generating" immediately, then "complete" or "error" on finish.

`POST /reels/{job_id}/approve/{step_name}` — Mark step as approved, advance current_step to next index. Per D-02: only advances forward. Uses `flag_modified(job, "step_state")` for SQLAlchemy JSON detection.

`POST /reels/{job_id}/regenerate/{step_name}` — Re-run step. Per D-02 and Research Pitfall 4: marks steps `step_name+1` through `video` as `approved: false` and clears their artifact fields. Then triggers step execution (same as POST /step/{step_name}).

`PUT /reels/{job_id}/edit/{step_name}` — Per D-03: update editable step artifacts inline.
- prompt: update `step_state.prompt.text`
- script: update `step_state.script.json` and job.script_json
- srt: reconstruct SRT file from entries, save to disk, update step_state.srt.path
- Other steps: return 400 (not editable per D-03)

`GET /reels/{job_id}/file/{filename}` — Serve artifact files via `FileResponse`. Resolves filename relative to job's `job_dir` (from step_state.prompt.job_dir or derived from job_id). Tenant-isolated. Per Research Pitfall 3: this enables frontend previews of images, audio, and video.

Add `STEP_ORDER = ["prompt", "images", "script", "tts", "srt", "video"]` as module-level constant.

For all step background tasks, use the `get_session_factory()` pattern (per Phase 999.1 decision). Load config_override from job.config_id same as existing `_generate_reel_task`.

Helper function `_get_user_job(job_id, user_id, db) -> ReelsJob` to DRY tenant-isolated job lookup (raises 404 if not found).
