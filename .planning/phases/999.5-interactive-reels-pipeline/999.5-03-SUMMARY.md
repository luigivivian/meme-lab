## Decisions Made
- StepperHeader detects completed steps by checking the `approved` field on each step's data object
- StepContent uses useRef to track previous step for slide direction calculation
- Approve-then-execute pattern: each step component calls approveStep then executeStep for the next step
- StepScript saves dirty edits automatically when user clicks approve to prevent data loss
- Non-null assertion (`stepState!`) used in [jobId] page after early return guard for TypeScript narrowing
