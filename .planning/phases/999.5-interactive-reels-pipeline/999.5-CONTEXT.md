# Phase 999.5: Interactive Reels Pipeline - Context

**Gathered:** 2026-03-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Transformar o pipeline de Reels (999.4) de fire-and-forget para fluxo interativo com stepper UI. Cada etapa requer aprovacao do usuario com opcao de regenerar/editar. Videos longos (>30s) sao segmentados em blocos consistentes e concatenados.

</domain>

<decisions>
## Implementation Decisions

### D-01: Stepper UI — 6 Etapas com Aprovacao
- Stepper visual moderno (horizontal ou vertical) com 6 etapas:
  1. **Prompt** — gerar/editar prompt das imagens → usuario revisa/edita → aprova
  2. **Imagens** — gerar imagens 9:16 → usuario ve grid, pode regenerar individual → aprova
  3. **Roteiro** — gerar roteiro JSON → usuario revisa/edita texto da narracao e legendas → aprova
  4. **Narracao** — gerar audio TTS → usuario ouve preview → aprova
  5. **Legendas** — gerar SRT → usuario revisa timings/texto → aprova
  6. **Video** — compor video final → preview in-browser → download/publicar
- Cada etapa tem botoes: "Aprovar", "Regenerar" (repete etapa), "Editar" (onde aplicavel)
- Progresso salvo no banco — usuario pode fechar e retomar depois
- UI intuitiva: cards grandes, previews inline, transicoes suaves

### D-02: Regeneracao Individual
- Regenerar qualquer etapa volta apenas para aquela etapa (nao reinicia tudo)
- Regenerar imagem individual: pode refazer 1 das N imagens sem perder as outras
- Regenerar roteiro: mantem imagens, refaz apenas texto
- Estado de cada etapa persistido no banco (JSON column no ReelsJob)

### D-03: Edicao Inline
- Prompt: textarea editavel antes de gerar imagens
- Roteiro: campos editaveis (narracao por cena, legenda_overlay, gancho, CTA)
- Legendas: editor de SRT simplificado (texto + timings)
- Narracao: nao editavel (regenerar com texto alterado)

### D-04: Videos Longos via Segmentacao
- Videos >30s sao divididos em segmentos de ~30s cada
- Cada segmento gera N imagens com mesmas refs/DNA (consistencia visual)
- Roteiro cobre todo o video mas e dividido em segmentos para geracao
- Audio TTS gera narracao completa de uma vez
- FFmpeg concatena segmentos com crossfade entre blocos
- Estrategia: roteiro primeiro (define segmentos), depois imagens por segmento

### D-05: Persistencia de Estado
- Tabela `reels_jobs` ja tem `script_json`, `image_paths`, `audio_path`, `srt_path`
- Adicionar coluna `step_state` JSON para persistir estado de cada etapa:
  ```json
  {
    "current_step": 2,
    "prompt": {"text": "...", "approved": true},
    "images": {"paths": [...], "approved": false},
    "script": {"json": {...}, "approved": false},
    "tts": {"path": "...", "approved": false},
    "srt": {"path": "...", "approved": false},
    "video": {"path": "...", "approved": false}
  }
  ```

### Claude's Discretion
- Layout do stepper (horizontal vs vertical, mobile responsivo)
- Animacoes de transicao entre etapas
- Como mostrar preview de audio (waveform simples ou play button)
- Editor de SRT: inline simples ou modal

</decisions>
