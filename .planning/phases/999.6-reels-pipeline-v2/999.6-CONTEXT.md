# Phase 999.6: Reels Pipeline v2 — Hailuo + Scene Context + Interactive Fix - Context

**Gathered:** 2026-03-28
**Status:** Ready for planning

<domain>
## Phase Boundary

Three fixes + one major feature upgrade to the reels pipeline:

1. **Interactive flow fix**: The /reels page currently calls the old sequential `/reels/generate` endpoint. The interactive `/reels/interactive` + step-by-step endpoints (from 999.5-02) exist but aren't wired to the frontend stepper. Fix: wire the "Criar Reel Interativo" button to POST /reels/interactive, and each step component to call executeStep/approveStep correctly so the user must approve before the next step runs.

2. **Per-scene image generation**: Currently image_gen.py generates N generic images with "Scene {i} of {n} — vary pose" which doesn't follow the script. Fix: reverse the order — generate script FIRST (step 2), then generate images per-cena using each cena's narracao/descricao as context. The stepper order becomes: prompt → script → images → narration → subtitles → video.

3. **Hailuo video generation**: Replace FFmpeg slideshow with Hailuo 2.3 Fast (768p) image-to-video via existing Kie.ai client. Each scene image gets animated by Hailuo with a motion prompt derived from the cena description. Final video = concatenated Hailuo clips with xfade transitions.

4. **Mobile subtitle styling**: Current FFmpeg subtitles filter uses FontSize=52 which is too large for 9:16 portrait. Fix subtitle styling to match meme style — smaller font, proper margins for mobile viewing.

</domain>

<decisions>
## Implementation Decisions

### Step Order Change
- Reorder stepper: prompt → script → images → tts → srt → video (was prompt → images → script)
- Script is generated from prompt text (no images needed as input)
- Images are generated per-cena AFTER script is approved, using cena context
- This enables contextual image generation that follows the story

### Hailuo Integration
- Use existing KieSora2Client from src/video_gen/kie_client.py
- Model: hailuo/2-3-image-to-video-standard with input_format "hailuo" (also works for Fast — same format)
- Actually use model_id for Hailuo 2.3 Fast: check config.py VIDEO_MODELS for exact ID
- Duration: 6s per scene clip (shortest Hailuo option)
- Upload scene images to GCS via existing GCSUploader for public URLs
- Build motion prompt per-cena using VideoPromptBuilder.build_motion_prompt()
- Poll each Hailuo task, download clips, concatenate with FFmpeg xfade
- Cost tracking via existing api_usage pattern

### Image Generation
- Keep Gemini for image generation but pass per-cena context
- Each image prompt includes: character DNA + cena narracao + scene description + position in story
- Generate 1 image per cena (not 5 generic images)
- User can regenerate individual scene images

### Subtitle Styling
- Reduce FontSize to ~28 for mobile
- Increase MarginV to position subtitles in lower safe zone
- Use outline style matching meme aesthetic (white text, black outline)

### Claude's Discretion
- Exact Hailuo model ID selection between Fast variants
- Motion prompt template for reels scenes
- Exact subtitle font size and positioning values
- How to handle Hailuo failures (retry vs fallback to FFmpeg)

</decisions>
