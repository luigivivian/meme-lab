# Phase 999.6: Reels Pipeline v2 — Hailuo + Scene Context + Interactive Fix - Research

**Researched:** 2026-03-28
**Domain:** Video pipeline (FFmpeg, Kie.ai Hailuo, Gemini image gen, FastAPI interactive stepper)
**Confidence:** HIGH

## Summary

This phase has four distinct work areas: (1) wiring the frontend interactive stepper to actually call the backend interactive endpoints instead of the sequential `/reels/generate`, (2) reordering the pipeline so script generates before images and each image uses per-cena context, (3) replacing the FFmpeg slideshow assembly with Hailuo image-to-video generation per scene via the existing Kie.ai client, and (4) fixing subtitle styling for mobile 9:16.

All building blocks already exist in the codebase. The Kie.ai client (`KieSora2Client`) handles Hailuo's `input_format: "hailuo"` with `image_url`, `duration: "6"`, and `resolution: "768P"`. The GCS uploader provides public URLs. The `VideoPromptBuilder.build_motion_prompt()` generates LLM-powered motion descriptions. The interactive endpoints (`POST /reels/interactive`, `POST /reels/{jobId}/step/{step}`, approve, regenerate, edit) are fully implemented. The frontend stepper components exist with approve/regenerate flows.

**Primary recommendation:** Reorder backend STEP_ORDER and pipeline methods first, then update frontend wiring, then implement Hailuo video step, and finally fix subtitles. Each change is independently testable.

### Anti-Patterns to Avoid
- **Don't generate images before script:** The whole point of v2 is context-aware images. Script must be approved first.
- **Don't poll Hailuo tasks sequentially:** Use `asyncio.gather` on multiple `poll_until_complete` calls for parallel polling.
- **Don't re-upload images that are already on GCS:** Store the public URL in step_state to avoid redundant uploads on regenerate.

## Common Pitfalls

### Pitfall 1: Script Step No Longer Needs Images
**What goes wrong:** The current `run_step_script()` takes `image_paths` as input and sends images to Gemini multimodal. In v2, script generates from prompt text only (before images exist).
**Why it happens:** Current `generate_script()` in script_gen.py is a multimodal call that reads images.
**How to avoid:** Create a new code path in `run_step_script` (or modify `generate_script`) that works with text-only input when no images are provided. Keep the multimodal path as fallback for the sequential (non-interactive) flow.
**Warning signs:** Script step fails because `image_paths` is empty.

### Pitfall 2: Step Order Mismatch Frontend/Backend
**What goes wrong:** Frontend stepper shows steps in wrong order, or approve triggers wrong next step.
**Why it happens:** Step order is defined in 3 places: `STEP_ORDER` (backend), `STEPS` array (stepper.tsx), and `renderStepContent` switch (jobId page). All must match.
**How to avoid:** Update all 3 locations atomically. The `current_step` index meaning changes (index 1 was "images", now "script").
**Warning signs:** Clicking "Aprovar" on script step triggers image generation (wrong step).

### Pitfall 3: Hailuo Task Timeout
**What goes wrong:** Hailuo tasks take 60-120s each. With 5 scenes, sequential polling = 5-10 minutes.
**Why it happens:** Default poll timeout is 600s but sequential polling wastes time.
**How to avoid:** Create all Hailuo tasks first, then poll all in parallel with `asyncio.gather`. Each task polls independently.
**Warning signs:** Video step takes >5 minutes for 5 scenes.

### Pitfall 4: GCS Signed URL Expiry During Hailuo Processing
**What goes wrong:** GCS signed URLs expire after 24h. Not a real risk for single runs, but if a job is paused and resumed days later, uploaded images may have expired.
**Why it happens:** Hailuo fetches the URL within seconds, so 24h is generous. But step_state stores the path, not the URL.
**How to avoid:** Store local path in step_state. Re-upload to GCS only when creating the Hailuo task (at video step time, not image step time).
**Warning signs:** Hailuo returns "image fetch failed" error.

### Pitfall 5: Image Count Mismatch
**What goes wrong:** Config says `image_count=5` but script has 4 cenas. Or script has 6 cenas but only 4 images generated.
**Why it happens:** In v2, image count = number of cenas in script. The old `image_count` config is irrelevant.
**How to avoid:** Generate exactly `len(cenas)` images. Don't use config `image_count` for per-scene generation.
**Warning signs:** IndexError when mapping images to cenas for Hailuo.

### Pitfall 6: reelFileUrl Path Resolution
**What goes wrong:** Frontend `reelFileUrl(jobId, path)` expects filename relative to job_dir, but step_state stores absolute paths.
**Why it happens:** `image_gen.py` returns absolute paths like `/Users/.../output/reels/.../images/image_00.jpg`.
**How to avoid:** The existing `serve_artifact_file` endpoint resolves relative to job_dir. Frontend must extract the relative path from the absolute path (currently done by splitting on job_dir). Keep this pattern.
**Warning signs:** 404s when previewing images in stepper.

### Hailuo Video Step

```python
async def run_step_video_hailuo(
    self,
    image_paths: list[str],
    audio_path: str,
    srt_path: str,
    job_dir: str,
    script: dict,
) -> str:
    """Generate per-scene Hailuo videos, concatenate with audio + subtitles."""
    from src.video_gen.kie_client import KieSora2Client
    from src.video_gen.gcs_uploader import GCSUploader
    from src.video_gen.video_prompt_builder import VideoPromptBuilder

    kie = KieSora2Client()
    gcs = GCSUploader()
    builder = VideoPromptBuilder()
    cenas = script.get("cenas", [])
    clips_dir = os.path.join(job_dir, "clips")
    os.makedirs(clips_dir, exist_ok=True)

    # Phase 1: Create all Hailuo tasks
    tasks = []
    for i, (img_path, cena) in enumerate(zip(image_paths, cenas)):
        url = gcs.upload_image(img_path, f"reels/{os.path.basename(job_dir)}/scene_{i}.jpg")
        motion = builder.build_motion_prompt(
            theme_key="generico",
            phrase_context=cena.get("narracao", ""),
            scene=cena.get("legenda_overlay", ""),
        )
        tasks.append((i, url, motion))

    # Phase 2: Submit all tasks to Hailuo
    task_ids = []
    for i, url, motion in tasks:
        task_id = await kie.create_task(
            image_url=url, prompt=motion, duration=6,
            model="hailuo/2-3-image-to-video-standard",
        )
        task_ids.append((i, task_id))

    # Phase 3: Poll all in parallel
    async def poll_and_download(idx, tid):
        result = await kie.poll_until_complete(tid)
        if result and result.video_url:
            clip_path = os.path.join(clips_dir, f"clip_{idx:02d}.mp4")
            await kie.download_video(result.video_url, clip_path)
            return clip_path
        return None

    clip_results = await asyncio.gather(
        *[poll_and_download(i, tid) for i, tid in task_ids]
    )
    clip_paths = [p for p in clip_results if p]

    # Phase 4: Concat clips with audio + subtitles
    video_path = os.path.join(job_dir, "final.mp4")

    ...
    return video_path
```

## Hailuo Model Selection (Claude's Discretion)

The CONTEXT.md mentions both "hailuo/2-3-image-to-video-standard" and "Hailuo 2.3 Fast 768p (R$1.05/video)".

Looking at `config.py` VIDEO_MODELS:
- `hailuo/2-3-image-to-video-standard`: R$1.31/6s, 720p, input_format "hailuo"

There is no explicit "Fast" variant in VIDEO_MODELS. The `bytedance/v1-lite-image-to-video` (Seedance Lite) costs R$1.05/5s but is a different model. The cheapest Hailuo option is Standard at R$1.31/6s.

**Recommendation:** Use `hailuo/2-3-image-to-video-standard` as configured. For a 5-scene reel: 5 x R$1.31 = R$6.55 per reel. This is the cheapest Hailuo option available.

## Hailuo Failure Handling (Claude's Discretion)

**Recommendation:** Retry once on transient errors (already handled by KieSora2Client's retry logic). On permanent failure for a scene, fall back to a static image clip (FFmpeg loop of the scene image for 6s) so the reel still completes. Log the failure for user visibility in step_state.

```python
if clip_path is None:
    # Fallback: static image for 6s
    fallback = os.path.join(clips_dir, f"clip_{idx:02d}.mp4")
    subprocess.run([
        "ffmpeg", "-y", "-loop", "1", "-t", "6",
        "-i", img_path, "-c:v", "libx264", "-pix_fmt", "yuv420p",
        fallback,
    ])
    clip_paths.append(fallback)
```

## Open Questions

   - What we know: Current `generate_script()` sends images to Gemini as multimodal input
